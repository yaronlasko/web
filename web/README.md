# World Cup 2026 — Office Pool Predictions

A static site of expected-points-optimal World Cup 2026 score picks, blending
**Polymarket** markets (70%) with an Elo / Dixon-Coles model (30%), plus a
Monte-Carlo qualification projection.

Live site: published with **GitHub Pages** (built by GitHub Actions — no build step
to run by hand).

## Files
- `index.html` — the whole site (vanilla JS, no framework). Loads `data.js`, so it
  also works opened directly via `file://`.
- `data.js` / `data.json` — the prediction bundle, regenerated by the pipeline
  (`python -m src.worldcup_predict`).

## Refreshing the predictions
The pipeline lives in the repo root (`src/worldcup_*.py`, `config.py`,
`data/worldcup/`). It is re-run automatically by **`.github/workflows/refresh.yml`**:

- **every 6 hours** on a schedule, and
- **on demand** — Actions tab → *Refresh predictions* → **Run workflow** (this is the
  "re-predict from the latest market" button), and
- on every push to `main`.

Each run executes `standings → fetch → predict` (fold in finished results, pull fresh
Polymarket odds, rebuild the picks) and redeploys this `web/` folder to Pages.

To refresh locally instead:
```bash
python -m src.worldcup_standings   # fold in finished games' results
python -m src.worldcup_fetch       # pull fresh Polymarket odds
python -m src.worldcup_predict     # rebuild web/data.js + web/data.json
git commit -am "refresh" && git push
```
